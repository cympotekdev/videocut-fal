# 自我進化

> 讓 Agent 從錯誤中學習，持續改進

## 快速使用

```
使用者: 記錄一下剛才的問題
使用者: 更新口誤辨識的規則
使用者: 這個教訓要記下來
```

## 更新位置

| 內容類型 | 目標檔案 | 範例 |
|---------|---------|------|
| 使用者畫像 | `CLAUDE.md` | 偏好、習慣 |
| 方法論 + 回饋 | `*/tips/*.md` | 規則、教訓 |

## 流程

```
使用者觸發（「剛才失敗了」、「記錄一下」）
    ↓
【自動】回溯上下文，找出問題點
    ↓
【自動】讀目標檔案全文，理解現有架構
    ↓
【自動】整合到正文對應位置（不是只往末尾加！）
    ↓
【自動】回饋記錄只記事件，不重複規則
    ↓
回報更新結果
```

**重點**：不要問「什麼問題」，直接從上下文分析！

## 更新原則

### ❌ 錯誤做法：往末尾加

```markdown
## 回饋記錄
### 2026-01-14
- 教訓：審查稿末尾一定要產生刪除任務清單
- 教訓：使用者確認時要分別確認口誤和靜音
```

只加到回饋記錄 = 規則散落在末尾，下次還是會犯錯

### ✅ 正確做法：整合到正文

1. **讀全文**，理解章節架構
2. **找到對應位置**，把規則整合進去
3. **回饋記錄只記事件**：`- 審查稿標記了靜音，但剪輯時漏刪`

```markdown
## 四、審查稿格式
（新增刪除任務清單範本）

## 五、確認與執行流程  ← 缺這個章節就新增
（新增分別確認口誤和靜音的流程）

## 回饋記錄
### 2026-01-14
- 審查稿標記了靜音，但剪輯時漏刪（只刪了口誤）
```

## 觸發條件

- 使用者糾正 AI 錯誤
- 使用者說「記住這個」、「以後注意」
- 發現新的通用規律

## 反例

### 2026-01-13
```
❌ 錯誤：
使用者: 剛才失敗了，更新到 skills
AI: 請告訴我你發現了什麼問題？  ← 不該問！

✅ 正確：
AI: [自動回溯上下文，找到失敗點]
AI: [執行更新]
```

### 2026-01-14
```
❌ 錯誤：
AI: 已更新，在回饋記錄新增 3 條教訓  ← 只加末尾！

✅ 正確：
AI: [讀全文，理解架構]
AI: [整合到正文對應位置]
AI: [回饋記錄只記事件]
AI: 已更新：新增第五章「確認與執行流程」，更新第四章範本
```

**原則**：規則要整合到正文，回饋記錄只是事件日誌
